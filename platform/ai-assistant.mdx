Prometheux includes an AI code assistant powered by the **Vadalog AI Agent** — a streaming, conversational interface that can autonomously plan and execute data workflows. Working inside a project, the agent can generate Vadalog programs, run concepts, inspect results, and iterate based on what it finds.

## What the agent can do

The agent maintains conversation history within a session, so follow-up messages have full context. In a single turn it can:

- Generate a new Vadalog concept from a natural-language request.
- Run concepts and read back their results.
- Propose concept code for you to review and accept.
- Iterate on a workflow based on the outcomes of the steps it ran.

As the agent works, it streams its activity back: text replies, the tools it is calling, the concept code it proposes, and the results of each step. This drives the tight feedback loop you see in the assistant chat.

For the full request/response contract (the `POST /agent/{project_id}/chat` streaming endpoint, the NDJSON event types such as `tool_start`, `proposal`, `tool_result`, `content`, and `done`, and how to reset a session), see the [Agent API reference](/integrations/api-endpoints/agent).

## Translation helpers

Alongside the conversational agent, Prometheux can translate other inputs into Vadalog so you don't have to write it by hand:

- **Natural language → Vadalog** — describe what you want in plain English. You can reference existing concepts with `@concept_name` so their schemas are used as context for the translation.
- **SQL → Vadalog** — convert a SQL query into an equivalent Vadalog program.
- **RDF → Vadalog** — turn RDF data into Vadalog.
- **OWL → Vadalog** — translate an OWL ontology into Vadalog rules (RDF/XML, Turtle, N3, and N-Triples), optionally saving the result as a concept.

These translations are LLM-powered. For request parameters and response shapes, see the [Vadalingo (Translations) API reference](/integrations/api-endpoints/vadalingo).
