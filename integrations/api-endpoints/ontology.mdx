---
title: "Ontology"
description: "Save and load a project's ontology, set concept ontology roles (node/edge), grow the lineage, describe it with the LLM, and import OWL ontologies."
---

An **ontology** is the semantic layer over a project's concepts — it labels each
concept as a graph **node** or **edge**, records the relationships between them,
and drives the ontology/lineage view. The Ontology API lets you persist and
reload the ontology document, assign ontology roles to individual concepts, add
elements to the lineage one at a time, generate a natural-language description,
and import an existing OWL file into the editor's JSON format.

These endpoints are mounted under the `/concepts` prefix. All paths are relative
to the [base URL](/integrations/rest-api#base-url) and require
[authentication](/integrations/api-endpoints/authentication). All responses use
the standard [envelope](/integrations/rest-api#response-envelope).

## Save the ontology

Persist a project's full ontology document (nodes and edges) for later reload.

```
POST /concepts/{project_id}/save-ontology
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `ontology_data` | body | yes | — | The ontology document — an object with `nodes` and `edges`, or a JSON string. |
| `scope` | body | no | `"user"` | [Scope](/integrations/rest-api#scope) of the project. |

```bash
curl -X POST "$BASE_URL/concepts/$PROJECT_ID/save-ontology" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "scope": "user",
    "ontology_data": {
      "nodes": [{ "id": "company", "label": "Company" }],
      "edges": [{ "source": "company", "target": "location", "label": "located_in" }]
    }
  }'
```

```jsonc
{
  "status": "success",
  "message": "Ontology for project my-project-001 saved successfully",
  "data": null
}
```

## Load the ontology

Return the ontology document for a project. `data` is `null` when the project
has no saved ontology yet.

```
GET /concepts/{project_id}/load-ontology
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `scope` | query | no | `"user"` | [Scope](/integrations/rest-api#scope) of the project. |

```bash
curl "$BASE_URL/concepts/$PROJECT_ID/load-ontology?scope=user" \
  -H "Authorization: Bearer $TOKEN"
```

```jsonc
{
  "status": "success",
  "message": "Ontology loaded successfully",
  "data": {
    "nodes": [{ "id": "company", "label": "Company" }],
    "edges": [{ "source": "company", "target": "location", "label": "located_in" }]
  }
}
```

## Set a concept's ontology type

Assign an ontology role to a single concept: mark it as a graph `node`, as an
`edge` (which requires `edge_source` and `edge_target`), or reset it to the
default by omitting `ontology_type`.

```
PATCH /concepts/{project_id}/concepts/{concept_name}/ontology-type
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `concept_name` | path | yes | — | The concept predicate name. |
| `ontology_type` | body | no | — | `"node"`, `"edge"`, or omit/`null` to reset to default. |
| `edge_source` | body | conditional | — | Source node predicate. **Required** when `ontology_type` is `"edge"`. |
| `edge_target` | body | conditional | — | Target node predicate. **Required** when `ontology_type` is `"edge"`. |
| `scope` | body | no | `"user"` | [Scope](/integrations/rest-api#scope) of the project. |

```bash
curl -X PATCH "$BASE_URL/concepts/$PROJECT_ID/concepts/located_in/ontology-type" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ontology_type": "edge",
    "edge_source": "company",
    "edge_target": "location",
    "scope": "user"
  }'
```

```jsonc
{
  "status": "success",
  "message": "Concept ontology type updated",
  "data": { "concept_name": "located_in", "ontology_type": "edge" }
}
```

## Add to lineage

Add a single ontology element — a node or an edge — to the project lineage
without rewriting the whole ontology document.

```
POST /concepts/{project_id}/add-to-lineage
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `element_type` | body | yes | — | `"node"` or `"edge"`. |
| `element_data` | body | yes | — | The element definition object. |
| `all_nodes` | body | no | `[]` | Current node list, used to resolve edge endpoints. |
| `scope` | body | no | `"user"` | [Scope](/integrations/rest-api#scope) of the project. |

```bash
curl -X POST "$BASE_URL/concepts/$PROJECT_ID/add-to-lineage" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "element_type": "node",
    "element_data": { "id": "supplier", "label": "Supplier" },
    "all_nodes": [],
    "scope": "user"
  }'
```

```jsonc
{
  "status": "success",
  "message": "Added to lineage: supplier",
  "data": { "predicate_name": "supplier" }
}
```

## Describe the ontology

Generate a short natural-language description of an ontology document using the
configured LLM.

```
POST /concepts/{project_id}/describe-ontology
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `ontology_data` | body | yes | — | The ontology document to describe. |
| `project.scope` | body | no | `"user"` | [Scope](/integrations/rest-api#scope), passed as `{ "project": { "scope": "user" } }`. |

```bash
curl -X POST "$BASE_URL/concepts/$PROJECT_ID/describe-ontology" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ontology_data": {
      "nodes": [{ "id": "company", "label": "Company" }],
      "edges": [{ "source": "company", "target": "location", "label": "located_in" }]
    },
    "project": { "scope": "user" }
  }'
```

```jsonc
{
  "status": "success",
  "message": "Ontology description generated successfully",
  "data": { "description": "This ontology models companies and the locations they operate in..." }
}
```

## Import an OWL ontology

Convert an OWL file into the ontology editor's JSON format (nodes and edges).
When `base_namespace` is omitted, the platform attempts to auto-detect it from
the OWL `xml:base` or `ontologyIRI` — if neither is present a `400` is returned
asking you to supply it explicitly.

```
POST /concepts/{project_id}/import-owl
```

| Parameter | In | Required | Default | Description |
|---|---|---|---|---|
| `project_id` | path | yes | — | The project ID. |
| `owl_content` | body | yes | — | The OWL ontology as a string (RDF/XML, Turtle, N3, or N-Triples). |
| `base_namespace` | body | no | auto-detected | Base namespace URI (e.g. `"http://example.org/onto#"`). Required if it cannot be auto-detected. |

```bash
curl -X POST "$BASE_URL/concepts/$PROJECT_ID/import-owl" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "owl_content": "<?xml version=\"1.0\"?><rdf:RDF ...>...</rdf:RDF>",
    "base_namespace": "http://example.org/onto#"
  }'
```

```jsonc
{
  "status": "success",
  "message": "OWL imported successfully. Found 4 nodes and 3 edges",
  "data": {
    "ontology_data": {
      "nodes": [{ "id": "Person", "label": "Person" }],
      "edges": [{ "source": "Person", "target": "Organization", "label": "worksAt" }]
    },
    "warnings": []
  }
}
```

<Tip>
To turn an OWL ontology directly into an executable Vadalog program (with
`@bind` annotations and optional Parquet output) rather than editor JSON, use
[OWL → Vadalog](/integrations/api-endpoints/vadalingo#owl-vadalog) instead.
</Tip>
