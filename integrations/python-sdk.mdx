The `prometheux_chain` Python SDK provides a high-level interface for working with all Prometheux platform features programmatically.

## Installation

```bash
pip install --upgrade prometheux_chain
```

## Configuration

```python
import prometheux_chain as px
import os

# Set your authentication token
os.environ['PMTX_TOKEN'] = 'YOUR_PMTX_TOKEN'

# Configure the backend URL
px.config.set('JARVISPY_URL', "https://api.prometheux.ai/jarvispy/{organization}/{username}")
```

## First API Call

```python
# Create a new project
project_id = px.save_project(project_name="my_first_project")
print(f"Created project: {project_id}")
```

## Response Format

All REST API responses follow a consistent format:

```json
{
  "data": {},           // Response data (varies by endpoint)
  "message": "string",  // Human-readable message
  "status": "success"   // Status: "success" or "error"
}
```

The Python SDK automatically handles responses and returns the relevant data directly.

## API Categories

The API is organized into the following categories:

| Category | Description |
|----------|-------------|
| [**Config**](/integrations/config) | SDK configuration and LLM setup |
| [**Data Sources**](/integrations/api-endpoints/data) | Data connections and schema inference |
| [**Projects**](/integrations/api-endpoints/projects) | Project management |
| [**Concepts**](/integrations/api-endpoints/concepts) | Concept execution and management |
| [**Vadalog**](/integrations/api-endpoints/vadalog) | Vadalog program evaluation |
| [**Dashboards**](/integrations/api-endpoints/dashboards) | Dashboard CRUD |
| [**Vadalingo**](/integrations/api-endpoints/vadalingo) | NL, SQL, RDF, and OWL to Vadalog translation |
| [**AI Agent**](/integrations/api-endpoints/agent) | Streaming conversational AI agent |

## Error Handling

The SDK raises exceptions for errors:

```python
import prometheux_chain as px

try:
    result = px.run_concept(project_id="invalid_id", concept_name="test")
except Exception as e:
    print(f"Error: {e}")
```
